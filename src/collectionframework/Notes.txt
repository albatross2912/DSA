Collection Framework
---------------------
1. Collection represents a single unit of objects i.e. a group.
2. Collections in javaÂ is a framework that provides an architecture to store and manipulate the group of objects.
3. All the operations that you perform on a data such as searching, sorting, insertion, manipulation, deletion etc. can be performed by Java Collections.

5. ArrayList class
    1. It is used to create Dynamic Array. so it doesn't require size.
    2. It allows Duplicate element
    3. It maintains Insertion order.

6. HashSet class
    1. It uses Hashtable to store elements.
    2. It uses hashing mechanism
    3. It maintains unordered collection.
    4. It does not allow duplicate elements.

9. Map Interface
    1. A Map contains values on the basis of key i.e. key and value pair.
    2. Each key and value pair is known as an entry.
    3. Map contains only unique keys.
    4. Map is useful if you have to search, update or delete elements on the basis of key.
    5. Map.Entry Interface
    6. Entry is the sub interface of Map. So we will be accessed it by Map.Entry ref_name
    7. It provides methods to get key and value.

10. HashMap class
